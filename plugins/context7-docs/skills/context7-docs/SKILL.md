# Context7 文档查询

当用户需要第三方库、框架或 SDK 的当前 API、配置方式、迁移说明或版本相关代码示例时使用本 Skill。

## 工作流

1. 先确认 `context7` MCP 依赖已经就绪；未就绪时使用 Skill 依赖安装链路。
2. 从用户问题和项目清单中确定库名、目标版本与具体任务；版本无法确定时说明所采用的版本范围。
3. 先解析准确的 Context7 library id，再用完整问题查询相关文档，避免只用宽泛关键词。
4. 将返回内容与仓库当前依赖版本对照后再给出实现建议，并指出任何版本不匹配。

## 约束

- Context7 聚合第三方资料；涉及安全、弃用或行为差异时，优先核对其返回的上游项目文档来源。
- 不要把其他版本的 API 直接套到当前项目。
- MCP 不可用时明确说明依赖状态；不要用 Shell 或 File 直接修改 MCP 配置来绕过安装审批。
